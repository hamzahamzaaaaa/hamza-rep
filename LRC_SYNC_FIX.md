# إصلاح مزامنة LRC - Pure Timestamp Mapping

## المشكلة
كان التطبيق يقوم بـ "العد العشوائي" للآيات عند:
- إيقاف الصوت (Pause)
- استئناف التشغيل (Resume)
- تغيير حالة المشغل

### السبب الجذري
كان الكود يستخدم **منطق Incremental** (Index++) بناءً على أحداث التشغيل بدلاً من الوقت الفعلي.

---

## ✅ الحل المطبق

### 1. المزامنة الزمنية الصرفة (Pure Timestamp Mapping)

```dart
void _syncWithPlayback(Duration position) {
  // Find verse based SOLELY on time comparison
  int newIndex = -1;
  for (int i = 0; i < _lrcEntries.length; i++) {
    if (_lrcEntries[i].timestamp <= position) {
      newIndex = i;  // هذا هو!
    } else {
      break;  // LRC مرتبة تصاعدياً
    }
  }
  
  // فقط تحديث إذا تغيرت الآية فعلياً
  if (newIndex == _activeVerseIndex) return;
}
```

### 2. إلغاء منطق Incremental (No Index++)

❌ **قبل (خاطئ)**:
```dart
if (isPlaying) {
  _currentVerseIndex++;  // خطأ! يعتمد على الأحداث
}
```

✅ **بعد (صحيح)**:
```dart
// الآية تُحسب ديناميكياً من الوقت فقط
currentVerse = lrcData.lastWhere(
  (v) => v.startTime <= currentPosition
);
```

### 3. الثبات عند التوقف

عند إيقاف الصوت عند **00:15**:
- يظل التظليل على الآية المرتبطة بـ **00:15** في ملف LRC
- لا يتحرك أبداً حتى يتغير الوقت الفعلي
- عند الاستئناف من نفس النقطة، يبقى على نفس الآية

## 📝 ملف LRC المرجعي

تم استخدام ملف LRC الفعلي لسورة مريم كمصدر نهائي:

```
[00:06.01]سورة مريم - سورة 19 - عدد آياتها 98
[00:11.66]بسم الله الرحمن الرحيم
[00:21.50]ذِكْرُ رَحْمَةِ رَبِّكَ عَبْدَهُ زَكَرِيَّا
[00:27.66]إِذْ نَادَى رَبَّهُ نِدَاء خَفِيًّا
...
```

### مزامنة مثال:
| الوقت | الآية |
|-------|-------|
| 00:11 | بسم الله الرحمن الرحيم |
| 00:21 | ذِكْرُ رَحْمَةِ رَبِّكَ عَبْدَهُ زَكَرِيَّا |
| 00:27 | إِذْ نَادَى رَبَّهُ نِدَاء خَفِيًّا |

إذا أوقفت عند **00:25** → يبقى على الآية الثانية (ذِكْرُ رَحْمَةِ...)

## 🔧 إصلاحات Parsing

### المشكلة السابقة
```dart
.replaceAll(RegExp(r'\d+'), '')  // ❌ يزيل جميع الأرقام!
```

هذا كان يزيل:
- أرقام الآيات ✓ (صحيح)
- **الأرقام من النص نفسه** ✗ (خطأ!)

### الحل الجديد
```dart
.replaceAll(RegExp(r'﴾\d+﴿'), '')  // إزالة ﴾40﴿ فقط
.replaceAll(RegExp(r'\(\d+\)'), '')  // إزالة (40) فقط
.replaceAll(RegExp(r'\[\d+\]'), '')  // إزالة [40] فقط
.replaceAll(RegExp(r'^\d+\s*'), '')  // إزالة الأرقام في البداية فقط
```

الآن يحافظ على الأرقام داخل النص العربي.

## 🎯 النتائج

### ✅ ما تم إصلاحه:
1. **المزامنة الزمنية الصرفة**: تعتمد فقط على `AudioPlayer.position`
2. **لا Index++**: لا يوجد زيادة تلقائية للآية
3. **الثبات عند التوقف**: الآية ثابتة طالما الوقت لم يتغير
4. **Parsing صحيح**: إزالة علامات الأرقام فقط، ليس كل الأرقام

### 📊 الاختبار:
```
إيقاف عند 00:15 → الآية ثابتة ✓
استئناف من 00:15 → نفس الآية ✓
الانتقال لـ 00:30 → ينتقل للآية الجديدة ✓
إيقاف عند 00:30 → الآية ثابتة ✓
```

---

## 🔍 كيف يعمل الآن

### عند التشغيل:
1. `AudioPlayer.position` يتغير كل ~250ms
2. `ref.listenManual` يستمع للتغييرات
3. `_syncWithPlayback(position)` يُستدعى
4. البحث عن آخر آية `timestamp <= position`
5. إذا تغيرت الآية → setState + glow animation

### عند الإيقاف:
1. `position` يتوقف عن التغير
2. `_syncWithPlayback` لا يُستدعى (لأن الوقت لم يتغير)
3. الآية تبقى كما هي → **ثبات تام**

### عند الاستئناف:
1. `position` يبدأ من حيث توقف
2. `_syncWithPlayback` يحسب الآية بناءً على الوقت الحالي
3. إذا كان الوقت لم يتغير → نفس الآية → **لا انتقال عشوائي**
